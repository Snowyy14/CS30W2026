Assignment Name: CountVowels

Understanding the Problem
How did you approach understanding the challenge?
Pretty simple concept - read a file and count the vowels. The main thing I had to figure out was how to actually open and read a file in Java since we haven't done a ton of file stuff before. The template helped a lot with showing me what classes to use.

Were there any parts of the problem you found confusing at first? If so, how did you resolve that confusion?
I was confused about why we need three different objects just to read a file (File, FileReader, BufferedReader). Seemed like overkill at first. But I get it now - File is just the path, FileReader opens it, and BufferedReader lets you read line by line which is way easier than character by character.

Planning the Solution
Did you create a plan or break the problem into smaller steps before coding?
Yeah, I thought about it in three steps: get the file name from the user, read through the file line by line, and check each character to see if its a vowel. Pretty straightforward once I broke it down.

How did you decide on the tools, data structures, or algorithms to use?
The template told me to use File, FileReader, and BufferedReader. For checking vowels I just used a bunch of if statements with equals(). I converted everything to lowercase first so I wouldn't have to check both uppercase and lowercase versions.

Implementation
Did you write the code in small pieces or attempt the entire solution at once?
I wrote the file reading part first without any vowel checking - just printed each line to make sure the file was being read right. Then I added the vowel checking logic inside the loop.

How did you test your solution along the way to make sure it was working?
I made a test file with some text I could manually count the vowels in. Like I typed "hello world" and counted 3 vowels by hand then ran my program to make sure it also got 3.

Overcoming Challenges
What part of the problem was the most difficult for you?
Getting the try-catch blocks right was annoying. I kept getting compiler errors about unhandled exceptions until I added catches for both FileNotFoundException and IOException.

How did you handle moments when you felt stuck or unsure of what to do next?
I looked at the template comments - they basically walked me through it step by step which was really helpful.

Learning
Was there anything you learned that you think will help you with future challenges?
The whole File -> FileReader -> BufferedReader pattern is something I'll definitely reuse. Once you learn it once it's pretty much the same every time. Also toLowerCase() is super handy for any kind of character comparison.
